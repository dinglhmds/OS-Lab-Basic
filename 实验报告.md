# 操作系统课程设计实验报告

## 基础必做部分：处理机调度、内存管理、进程同步与文件系统

---

## 一、实验目的

根据《操作系统》课程设计要求，独立完成基础必做部分（占70%），具体目标如下：

1. **处理机调度**：模拟实现 FCFS、SJF（非抢占/抢占）、RR、Priority（非抢占/抢占）等经典调度算法，支持动态输入和性能分析。
2. **内存管理**：模拟实现 FIFO/LRU 页面置换算法和首次适应(FF)/最佳适应(BF)动态分区管理，展示分配回收过程。
3. **进程同步与并发控制**：使用 POSIX 线程（pthread）实现生产者-消费者、读者-写者、哲学家进餐三个经典同步问题。
4. **文件系统**：设计实现简化版类 Unix 文件系统，支持文件/目录的创建、读写、删除及空闲空间管理。
5. **培养工程能力**：熟悉 Linux 开发环境、C99 系统级编程、Makefile 构建和代码规范。

### 4.2 模块一：处理机调度（scheduler/）

#### 4.2.1 数据结构设计

```c
typedef struct {
    int pid;                    // 进程标识
    char name[MAX_NAME_LEN];    // 进程名称
    float arrival_time;         // 到达时间
    float burst_time;           // 执行时间
    int priority;               // 优先级（数值越小优先级越高）
    float start_time;           // 首次开始时间
    float finish_time;          // 完成时间
    float turnaround_time;      // 周转时间 = 完成 - 到达
    float waiting_time;         // 等待时间 = 周转 - 执行
    float remaining_time;       // 剩余时间（用于抢占式算法）
    int completed;              // 是否已完成
} PCB;
```

#### 4.2.2 实现的算法

| 算法 | 类型 | 核心逻辑 |
|:---|:---|:---|
| **FCFS** | 非抢占 | 按到达时间排序，顺序执行 |
| **SJF 非抢占** | 非抢占 | 每次选就绪队列中 burst_time 最短的进程 |
| **SJF 抢占** | 抢占 | 每个时间单位重新检查，选 remaining_time 最短的进程 |
| **RR** | 抢占 | 时间片轮转，quantum=2，到期强制切换 |
| **Priority 非抢占** | 非抢占 | 按优先级排序，高优先级优先 |
| **Priority 抢占** | 抢占 | 每个时间单位重新检查，选优先级最高的进程 |

#### 4.2.3 关键实现细节

**甘特图生成**：使用 `GanttSlot` 数组记录每个调度段，支持合并连续同 PID 的段：
```c
typedef struct {
    int pid;
    float start;
    float end;
} GanttSlot;
```

**性能指标计算**：
- 周转时间 = `finish_time - arrival_time`
- 等待时间 = `turnaround_time - burst_time`
- 平均指标 = 总和 / 进程数

**测试用例**：内置 3 组预设测试用例，覆盖不同特征的任务集：
- Case 1：4 个进程，有明确的到达时间差异
- Case 2：5 个进程，混合短作业和长作业
- Case 3：3 个进程，同时到达，测试优先级效果

### 4.3 模块二：内存管理（memory/）

#### 4.3.1 页面置换（paging.c）

**数据结构**：
```c
typedef struct {
    int page_id;        // 页面号
    int frame_id;       // 物理帧号
    int valid;          // 是否在内存中
    int load_time;      // 装入时间（FIFO用）
    int last_access;    // 最后访问时间（LRU用）
} PageEntry;
```

**FIFO 算法**：使用循环队列记录页面装入顺序，淘汰最早进入的页面。

**LRU 算法**：记录每个页面的最后访问时间戳，淘汰最久未访问的页面。

**测试方法**：使用经典页面访问串：
- Test 1：`7 0 1 2 0 3 0 4 2 3 0 3 2 1 2 0 1 7 0 1`
- Test 2：`1 2 3 4 2 1 5 6 2 1 2 3 7 6 3 2 1 2 3 6`

#### 4.3.2 动态分区管理（partition.c）

**数据结构**：
```c
typedef struct Partition {
    int start;          // 起始地址
    int size;           // 分区大小
    int is_free;        // 是否空闲
    int pid;            // 占用进程号（-1表示空闲）
    struct Partition *next;
} Partition;
```

**首次适应（FF）**：从链表头部开始遍历，找到第一个足够大的空闲分区。

**最佳适应（BF）**：遍历全部空闲分区，找到能满足要求的最小分区。

**分区回收与合并**：释放分区时，自动检查并合并相邻的空闲分区。

### 4.4 模块三：进程同步（sync/）

#### 4.4.1 生产者-消费者（producer_consumer.c）

**同步机制**：
- `empty_slots` 信号量：记录缓冲区空位数量（初值 = BUFFER_SIZE）
- `filled_slots` 信号量：记录已填充位置数量（初值 = 0）
- `mutex` 互斥锁：保护缓冲区临界区

**实现参数**：
- 缓冲区大小：10
- 生产者数量：2，每个生产 5 个物品
- 消费者数量：2，每个消费 5 个物品

#### 4.4.2 读者-写者（readers_writers.c）

**同步机制**：
- `mutex`：保护读者计数器
- `rw_mutex`：读写互斥锁
- 读者优先策略：多个读者可同时读，写者需等待所有读者完成

**实现参数**：
- 读者数量：3
- 写者数量：2

#### 4.4.3 哲学家进餐（philosophers.c）

**死锁避免方案**：
- 5 位哲学家，5 支筷子（互斥锁）
- **奇偶号策略**：奇数号哲学家先拿左筷子再拿右筷子，偶数号相反
- 该策略打破了循环等待条件，避免死锁

**实现参数**：
- 哲学家数量：5
- 每位进餐 3 次
- 随机思考/进餐时间（100-300ms）

### 4.5 模块四：文件系统（filesystem/）

#### 4.5.1 核心数据结构

```c
// 超级块
typedef struct {

    int block_count;        // 数据块总数
    int free_inodes;        // 空闲 inode 数

// Inode（索引节点）
typedef struct {

    int size;               // 文件大小
    int blocks[DIRECT_BLOCKS];  // 10个直接块指针

// 目录项
typedef struct {
    char name[NAME_LEN];    // 文件名

```

### 5.2 模块一：处理机调度测试结果

#### 测试用例 1（4 个进程）

| 进程 | 到达时间 | 执行时间 | 优先级 |
|:---|:---|:---|:---|
| P1 | 0 | 8 | 2 |
| P2 | 1 | 4 | 1 |
| P3 | 2 | 9 | 4 |
| P4 | 3 | 5 | 3 |

**各算法性能对比**：

| 算法 | 平均周转时间 | 平均等待时间 | 特点 |
|:---|:---|:---|:---|
| **FCFS** | 15.25 | 8.75 | 简单，护航效应明显 |
| **SJF 非抢占** | 14.25 | 7.75 | 短作业优先，整体较优 |
| **SJF 抢占** | **13.00** ⭐ | **6.50** ⭐ | **最优**，抢占减少等待 |
| **RR (q=2)** | 19.25 | 12.75 | 公平但切换开销大 |
| **Priority 非抢占** | 15.25 | 8.75 | 高优先级先执行 |
| **Priority 抢占** | 15.25 | 8.75 | 与 non-preemptive 相同（无更高优先级到达） |

**分析**：
- SJF 抢占式在该测试用例上表现最优，因为短作业（P2 burst=4, P4 burst=5）能够被及时调度
- RR 公平性最好（每个进程都能定期获得 CPU），但频繁上下文切换导致周转时间增加
- FCFS 实现最简单，但对短作业不友好（P4 等待了 18 个时间单位）

#### 测试用例 3（3 个进程同时到达）

| 算法 | 平均周转时间 | 平均等待时间 |
|:---|:---|:---|
| FCFS | 16.00 | 8.33 |
| **SJF 非抢占** | **13.67** | **6.00** |
| **SJF 抢占** | **13.67** | **6.00** |
| RR (q=2) | 19.67 | 12.00 |
| Priority 非抢占 | 16.00 | 8.33 |
| Priority 抢占 | 16.00 | 8.33 |

**分析**：同时到达时，SJF 的优势更加明显（短作业 X=5, Y=8, Z=10），而 Priority 和 FCFS 效果相同（按 PID 顺序）。

## 六、关键代码规范

### 6.1 编译标准

所有模块统一使用严格的编译选项：
```makefile
CFLAGS := -Wall -Wextra -Werror -std=c99 -O2 -I../common -D_POSIX_C_SOURCE=200809L
```

- `-Wall -Wextra`：开启所有警告
- `-Werror`：将警告视为错误，强制代码规范
- `-std=c99`：C99 标准
- `-O2`：优化级别 2

### 6.2 错误处理

所有 `malloc` 和系统调用均检查返回值：
```c
static inline void *safe_malloc(size_t size) {
    void *p = malloc(size);
    if (!p && size > 0) PANIC("malloc(%zu) failed", size);
    return p;
}
```

### 6.3 代码复用

通过 `common/` 公共库实现了代码复用：
- 调度器和内存管理模块复用 `list.c` 的链表实现
- 所有模块复用 `utils.c` 的输入输出工具
- 统一的颜色输出和错误处理风格

## 七、遇到的问题与解决方案

| 问题 | 原因 | 解决方案 |
|:---|:---|:---|
| `usleep` 函数未声明 | C99 标准下 `usleep` 需要 `_POSIX_C_SOURCE` 宏 | 在 Makefile 中添加 `-D_POSIX_C_SOURCE=200809L` |
| `scanf` 返回值被忽略 | `-Werror` 将警告视为错误 | 所有 `scanf` 调用添加返回值检查：`if (scanf(...) != 1) { WARN(...); return 1; }` |
| 哲学家进餐死锁 | 所有哲学家同时拿左筷子，形成循环等待 | 采用奇偶号策略：奇数号先拿左筷子，偶数号先拿右筷子 |
| 文件系统目录项解析错误 | `sscanf` 处理 `write` 命令时空格分隔问题 | 使用 `strchr` 手动分割文件名和数据内容 |
